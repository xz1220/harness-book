# 推广素材与文案

这个目录用于沉淀《Harness Book》的推广选题、短文案、视觉素材说明和发布版本。

## 目录约定

- `xiaohongshu/`：小红书风格选题、封面文案、正文草稿。
- `assets/`：配图、截图、封面设计稿等素材说明。实际图片文件可按平台和日期命名。

## 文案原则

推广内容优先从 user story 进入，而不是从概念定义进入。

一个好的选题先回答：

1. 谁在什么场景下遇到了问题？
2. 他原本以为问题是什么？
3. 为什么这个误解会导致失败？
4. Harness 这个概念如何重新解释这个问题？

比如：

> 面试官问：“用过 Harness 吗？”
>
> 候选人以为这是提示词工程。
>
> 但真正的问题是：当 Agent 要调用工具、处理权限、压缩上下文、避免死循环时，谁在替模型把这些工程判断执行起来？

这类内容的目标不是先解释完整定义，而是让读者意识到：Harness 不是 prompt，也不只是 framework，而是让 AI Agent 真正落地的那层系统工程。
